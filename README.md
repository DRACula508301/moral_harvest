# moral_harvest

PPO in Melting Pot Harvest with single-agent and multi-agent training modes, including reward-shaped multi-agent runs.

## Use Linux / Start WSL

This project should be run from Linux (or WSL on Windows), using the Linux virtual environment.

From Windows PowerShell:

```powershell
wsl
```

Then in WSL:

```bash
cd /mnt/c/Users/dchen/Documents/Projects/moral_harvest
```

Use:

```bash
.venv-linux/bin/python ...
```

## Training

### Minimal example (defaults only)

This runs with default training flags (single-agent, CleanRL backend, default substrate/agent/hyperparameters):

```bash
cd /mnt/c/Users/dchen/Documents/Projects/moral_harvest && .venv-linux/bin/python -m moral_harvest.cli.train
```

### Available training flags

Command:

```bash
.venv-linux/bin/python -m moral_harvest.cli.train [FLAGS]
```

Environment / run control:

- `--mode {single-agent,multi-agent-selfish,multi-agent-reward-shaped}` (default: `single-agent`)
- `--backend {rllib,cleanrl}` (default: `cleanrl`)
- `--substrate` (default: `commons_harvest__open`)
- `--focal-agent` (default: `player_0`; used in single-agent mode)
- `--num-agents` (default: `10`; used in multi-agent modes)
- `--num-envs` (default: `1`; vectorized environments)
- `--stop-iters` (default: `1000`)
- `--checkpoint-every` (default: `100`)
- `--checkpoint-root` (default depends on mode)
- `--results-root` (default depends on mode)
- `--run-name` (default: autogenerated)

Reward shaping (used in `multi-agent-reward-shaped` mode):

- `--reward-type {selfish,utilitarian,deontological,virtue,all}` (default: `utilitarian`)
- `--reward-alpha` (default: `0.5`), used in: `alpha*own + (1-alpha)*shaping`
- `--deontological-max-bonus` (default: `1.0`)
- `--virtue-scale` (default: `1.0`)

Optimization / PPO:

- `--num-env-runners` (default: `0`; legacy/unused for current CleanRL paths)
- `--train-batch-size` (default: `4000`)
- `--minibatch-size` (default: `256`)
- `--num-sgd-iter` (default: `10`; deprecated alias for `--num-epochs`)
- `--num-epochs` (default: unset; falls back to `--num-sgd-iter`)
- `--lr` (default: `3e-4`)
- `--gamma` (default: `0.99`)
- `--gae-lambda` (default: `0.95`)
- `--clip-coef` (default: `0.2`)
- `--clip-vloss` / `--no-clip-vloss` (default: enabled)
- `--ent-coef` (default: `0.01`)
- `--vf-coef` (default: `0.5`)
- `--target-kl` (default: `None`)
- `--max-grad-norm` (default: `0.5`)
- `--anneal-lr` / `--no-anneal-lr` (default: enabled)
- `--normalize-rgb` / `--no-normalize-rgb` (default: enabled)
- `--num-gpus` (default: `None`, auto-detect)
- `--seed` (default: `None`)

Wrapper controls:

- `--include-ready-to-shoot` (default: disabled; current CleanRL trainers expect RGB-only and may reject this)
- `--no-op-action` (default: `0`)

### Expected training output and locations

Console output:

- Iteration logs with reward and loss metrics.
- Periodic `checkpoint_saved=...` lines.
- Final JSON summary (includes mode/backend/run name/final checkpoint).

Artifacts:

- Metrics: `<results_root>/cleanrl/<run_name>/metrics.jsonl`
- Metrics CSV: `<results_root>/cleanrl/<run_name>/metrics.csv`
- Plot: `<results_root>/cleanrl/<run_name>/training_curves.png` (generated when the command returns a single run)
- Checkpoints (single-agent default): `checkpoints/single_agent/...`
- Checkpoints (multi-agent selfish default): `checkpoints/multi_agent/selfish/...`
- Checkpoints (reward-shaped default): `checkpoints/<reward_type>/...`

Notes:

- In `multi-agent-reward-shaped` with `--reward-type all`, training runs each reward type sequentially.
- For `--reward-type all`, output is aggregated across runs and per-run plots are not auto-generated by the current CLI.

## Rollout (single-agent checkpoints)

### Minimal rollout example

```bash
cd /mnt/c/Users/dchen/Documents/Projects/moral_harvest && .venv-linux/bin/python -m moral_harvest.cli.rollout_single_agent --checkpoint-path <PATH_TO_CHECKPOINT>
```

`<PATH_TO_CHECKPOINT>` can be:

- CleanRL checkpoint file (`.pt`)
- RLlib checkpoint directory (if you already have one)

### Available rollout flags

Command:

```bash
.venv-linux/bin/python -m moral_harvest.cli.rollout_single_agent [FLAGS]
```

- `--checkpoint-path` (required)
- `--backend {auto,rllib,cleanrl}` (default: `auto`; `.pt` => `cleanrl`, otherwise `rllib`)
- `--episodes` (default: `5`)
- `--substrate` (default: `commons_harvest__open`)
- `--focal-agent` (default: `player_0`)
- `--include-ready-to-shoot` (default: disabled)
- `--no-op-action` (default: `0`)
- `--explore` (default: disabled; if enabled, samples stochastically)
- `--seed` (default: `None`)
- `--record-video` (default: disabled)
- `--run-name` (default: auto-generated from checkpoint name; used only when recording video)

### Expected rollout output and locations

Console output:

- Per-episode lines: `episode=... steps=... total_reward=...`
- Final JSON summary with `mean_reward`, `min_reward`, `max_reward`, and per-episode results.

Artifacts:

- If `--record-video` is set, videos are saved under `videos/<run_name>/`.
- If `--record-video` is not set, rollout writes no files by default.
